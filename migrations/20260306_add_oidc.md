# Migration: 20260306_add_oidc

## Summary

Adds full OIDC Provider support to the database schema. This migration extends the `applications` table with per-app OIDC settings and creates two new tables — `oidc_clients` and `oidc_auth_codes` — that form the persistence layer for the OIDC feature.

## Schema Changes

### 1. `applications` — new OIDC columns

| Column | Type | Default | Description |
|--------|------|---------|-------------|
| `oidc_enabled` | `BOOLEAN` | `FALSE` | Master switch — expose OIDC endpoints for this app |
| `oidc_rsa_private_key` | `TEXT` | `''` | PKCS#8 PEM RSA-2048 private key for RS256 ID-token signing. Generated on first use. Never returned by the API. |
| `oidc_id_token_ttl` | `INTEGER` | `3600` | ID token lifetime in seconds (default 1 hour) |
| `oidc_issuer_url` | `VARCHAR(500)` | `''` | Optional custom issuer URL. Empty = auto-generated from `PUBLIC_URL + /oidc/<app_id>` |

All columns use `ALTER TABLE … ADD COLUMN IF NOT EXISTS`, so the statement is idempotent if GORM AutoMigrate already created them on first boot.

---

### 2. `oidc_clients` (new table)

Stores registered OIDC/OAuth2 relying-party client applications. Each client is scoped to an `Application` via `app_id`.

**Primary key:** `id UUID`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, default `gen_random_uuid()` | Internal identifier |
| `app_id` | `UUID` | NOT NULL, FK → `applications(id)` ON DELETE CASCADE | Owning application |
| `name` | `VARCHAR(100)` | NOT NULL | Display name on consent screen |
| `description` | `TEXT` | NOT NULL, default `''` | Optional description |
| `client_id` | `VARCHAR(64)` | NOT NULL, UNIQUE | Public client identifier |
| `client_secret_hash` | `TEXT` | NOT NULL | bcrypt hash of the client secret |
| `redirect_uris` | `TEXT` | NOT NULL, default `'[]'` | JSON array of registered redirect URIs |
| `allowed_grant_types` | `VARCHAR(200)` | NOT NULL | Comma-separated grant types |
| `allowed_scopes` | `VARCHAR(200)` | NOT NULL | Comma-separated scopes |
| `require_consent` | `BOOLEAN` | NOT NULL, default `TRUE` | Show consent screen? |
| `is_confidential` | `BOOLEAN` | NOT NULL, default `TRUE` | Has a client secret? |
| `pkce_required` | `BOOLEAN` | NOT NULL, default `FALSE` | Enforce PKCE? |
| `is_active` | `BOOLEAN` | NOT NULL, default `TRUE` | Soft-disable without deletion |
| `logo_url` | `TEXT` | NOT NULL, default `''` | Optional logo URL for consent screen |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, default `NOW()` | |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, default `NOW()` | |

**Indexes:**

| Index | Columns | Purpose |
|-------|---------|---------|
| `uq_oidc_clients_client_id` | `client_id` | Unique — fast lookup by public client ID |
| `idx_oidc_clients_app_id` | `app_id` | List clients per application |
| `idx_oidc_clients_app_id_active` | `(app_id, is_active)` | Authorization endpoint — find active clients |

**Foreign keys:**

- `app_id` → `applications(id)` ON DELETE CASCADE — deleting an application removes all its OIDC clients

### 3. `oidc_auth_codes` (new table)

Stores single-use authorization codes issued during the Authorization Code flow. Each code is consumed at the `/token` endpoint and cannot be replayed.

**Primary key:** `id UUID`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Internal identifier |
| `app_id` | `UUID` | NOT NULL | Owning application |
| `client_id` | `VARCHAR(64)` | NOT NULL | Client that initiated the request |
| `user_id` | `UUID` | NOT NULL | End user who approved the request |
| `code` | `VARCHAR(128)` | NOT NULL, UNIQUE | The random single-use code |
| `redirect_uri` | `TEXT` | NOT NULL | Must match the value used in the token request |
| `scopes` | `TEXT` | NOT NULL | Space-separated granted scopes |
| `nonce` | `TEXT` | NOT NULL, default `''` | Echoed into the ID token |
| `code_challenge` | `TEXT` | NOT NULL, default `''` | PKCE code challenge |
| `code_challenge_method` | `VARCHAR(10)` | NOT NULL, default `''` | `"S256"` or empty |
| `expires_at` | `TIMESTAMPTZ` | NOT NULL | Code validity window |
| `used` | `BOOLEAN` | NOT NULL, default `FALSE` | Replay protection flag |
| `created_at` | `TIMESTAMPTZ` | NOT NULL | |

**Indexes:**

| Index | Columns | Purpose |
|-------|---------|---------|
| `uq_oidc_auth_codes_code` | `code` | Unique — fast lookup during token exchange |
| `idx_oidc_auth_codes_app_id` | `app_id` | Filter by application |
| `idx_oidc_auth_codes_client_id` | `client_id` | Lookup by client |
| `idx_oidc_auth_codes_expires_at` | `expires_at` | Expiry cleanup worker |
| `idx_oidc_auth_codes_client_used` | `(client_id, used)` | Token exchange — find unused codes by client |

## Application Configuration

The OIDC feature is gated by environment variable:

```
OIDC_ENABLED=true          # default: false
PUBLIC_URL=https://auth.example.com
OIDC_ID_TOKEN_EXPIRATION_MINUTES=60      # default: 60
OIDC_AUTH_CODE_EXPIRATION_MINUTES=10     # default: 10
```

---

## OIDC Endpoints (registered when OIDC_ENABLED=true)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/.well-known/openid-configuration` | Global discovery redirect |
| GET | `/oidc/:app_id/.well-known/openid-configuration` | Per-app discovery document |
| GET | `/oidc/:app_id/.well-known/jwks.json` | RSA public JWKS |
| GET | `/oidc/:app_id/authorize` | Authorization endpoint |
| POST | `/oidc/:app_id/authorize` | Consent form submit |
| POST | `/oidc/:app_id/token` | Token endpoint |
| GET/POST | `/oidc/:app_id/userinfo` | User info claims |
| POST | `/oidc/:app_id/introspect` | Token introspection |
